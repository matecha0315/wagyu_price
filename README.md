# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

リポジトリとはデータの保管場所。データの変更履歴も保存される。

ローカルリポジトリとはPC上に作成するリポジトリで、開発者が個人で使うもの。

リモートリポジトリとはネット上に作成するリポジトリで、複数人で使うことができる。代表的なサービスはGithub。

## プッシュとマージの違いは何でしょうか？

プッシュはローカルリポジトリの変更をリモートリポジトリに反映すること。
マージはブランチからブランチに変更内容を取り込むこと。

## コミットとプッシュの違い

コミットは変更内容をローカルリポジトリに保存すること。
プッシュはローカルリポジトリの作業内容をリモートリポジトリに反映すること。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

簡潔に、変更内容が分かるように。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

プルリクエストを作成せずにマージすると変更が即反映される。開発現場ではふつう、幹となるブランチが本番の環境で使用されるため、十分な確認を行わずマージすることにはリスクがある。
プルリクエストを作成し、他者に変更内容を確認してもらうことで問題のあるコードが本番環境に反映される事態を防ぐことに繋がる。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

コンフリクトが起きた場合、どちらか一方の変更内容を選んでマージするか、コードを書き換えて両方の変更内容を取り込むか選ぶ必要がある。
適切な対応を見極めるために、コンフリクト発生箇所のコードの正しい理解が求められる。
